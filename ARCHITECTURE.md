# myHabbit — архітектура проєкту

## 1. Призначення
myHabbit — приватна сімейна PWA для 2–5 учасників. Система перетворює корисні справи на квести, розвиває особисті навички, видає досягнення та дозволяє витрачати зароблені монети у спільному магазині реальних можливостей.

## 2. Принципи
- Один Cloudflare Worker обслуговує API, PWA та десктопний лендінг.
- Немає зовнішньої бази продуктів або запитів до Open Food Facts.
- Калькулятор калорій працює ручним введенням.
- Дані сім’ї ізольовані кодом, PIN та токенами учасників.
- Максимальна сім’я — 5 профілів.
- Інтерфейс однаково придатний для телефона, встановленої PWA та ПК.
- RPG-механіки підтримують реальне життя, а не замінюють його віртуальними предметами.

## 3. Компоненти

### public/index.html
Мінімальна HTML-оболонка. Підключає стилі, PWA-маніфест і клієнтський застосунок.

### public/styles.css
Повна адаптивна дизайн-система:
- десктопний лендінг;
- бічна навігація для ПК;
- нижня навігація для мобільної PWA;
- картки квестів, навичок, профілів, ачивок і магазину;
- модальні вікна створення квестів та нагород.

### public/app.js
Клієнтський SPA без фреймворків і npm-залежностей.
Основні модулі:
- маршрутизація екранів;
- авторизація сім’ї;
- профілі учасників;
- квести;
- навички;
- досягнення;
- магазин;
- сімейна активність;
- ручний калькулятор калорій;
- локальне резервне збереження;
- синхронізація з Cloudflare Worker.

### public/sw.js
Service Worker для встановлення та кешування PWA.

### public/manifest.webmanifest
Назва, тема, іконки, категорії та швидкі переходи PWA.

### src/worker.js
Cloudflare Worker і Durable Object.
Відповідає за:
- роздачу статичних файлів;
- створення сім’ї;
- приєднання за кодом і PIN;
- видачу окремого токена кожному профілю;
- читання та синхронізацію спільного стану;
- обмеження сім’ї до 5 учасників;
- сумісність із наявною Telegram-інфраструктурою.

## 4. API сім’ї

### POST /api/family/create
Створює сім’ю та профіль власника.
Вхід: familyName, name, gender, pin.
Вихід: token, userId, familyCode, state.

### POST /api/family/join
Додає нового учасника.
Вхід: code, pin, name, gender.
Вихід: token, userId, familyCode, state.

### GET /api/family/state
Повертає актуальний спільний стан. Потребує Bearer-токен.

### PUT /api/family/state
Зберігає актуальний спільний стан. Потребує Bearer-токен.

## 6. Правила економіки
- Монети видаються лише за завершені квести.
- XP підвищує особистий та сімейний прогрес.
- Завдання також підвищують конкретну навичку.
- Магазин містить реальні можливості: клуб, ігри, техніку, подорожі, відпочинок.
- Запас товару може закінчуватися.
- Великі покупки можуть бути спільними фондами.
- Адміністратор сім’ї створює квести та позиції магазину.

## 7. Потік синхронізації
1. Користувач створює сім’ю або входить за кодом.
2. Worker видає персональний токен.
3. Клієнт отримує сімейний стан.
4. Кожна зміна спочатку зберігається локально.
5. Через коротку затримку стан надсилається в Durable Object.
6. Інші пристрої отримують оновлення при наступному відкритті або оновленні.

## 8. Безпека і масштаб
Для сімейного MVP використано просту модель повного синхронізованого стану. Вона оптимальна для 2–5 людей і не потребує окремої бази даних.

Перед публічним масштабуванням потрібно додати:
- розподіл ролей owner/admin/member;
- серверну перевірку кожної економічної операції;
- журнал транзакцій;
- optimistic locking за revision;
- відновлення токена;
- резервні копії та експорт сімейних даних.

## 9. Розгортання
Проєкт готовий до наявного GitHub Actions + Wrangler процесу.
Основні команди локально:

```bash
npx wrangler dev
npx wrangler deploy
```

Секрети Telegram залишаються сумісними, але не потрібні для базової роботи myHabbit.

## Брендинг і розгортання myHabbit

- Публічна назва продукту: **myHabbit**.
- GitHub-репозиторій: **myHabbit**.
- Технічне ім’я Cloudflare Worker: **myhabbit**.
- Telegram Bot API використовується тільки через зашифрований секрет `TELEGRAM_BOT_TOKEN`.
- Під час GitHub Actions workflow виконує `getMe`, отримує фактичний username бота, генерує секрет webhook, розгортає Worker і налаштовує Telegram Mini App.
- Токени, PIN-коди та webhook-секрети не повинні потрапляти до Git-репозиторію.

## Admin & Telegram tracking layer (v2.2)

### Roles
- Перший учасник сім'ї є адміністратором за замовчуванням.
- Поле `role` у профілі: `admin` або `member`.
- Лише адміністратор бачить розділ `Адмін` і кнопки створення контенту.

### Admin panel
- Dashboard здоров'я сімейного простору.
- CRUD-керування квестами: створення, пауза/активація, видалення.
- CRUD-керування магазином: створення, контроль залишку, видалення.
- Огляд кількості учасників, Telegram-підключень і сімейного фонду.

### Profiles
- Каталог учасників у розділі `Сім'я`.
- Клік по учаснику відкриває його рівень, навички, активність і колекцію досягнень.
- Telegram-статус відображається без показу приватних Telegram ID.

### Telegram connection
- `/api/telegram/config` використовується для перевірки бота та отримання username.
- Кнопка підключення відкриває бота з параметром `startapp=family_<family-code>`.
- Токен не зберігається у frontend, ZIP або Git.
