# myHabbit 2.0

Приватна сімейна PWA для 2–5 людей: спільні квести, особисті навички, досягнення та магазин реальних можливостей.

## Готові екрани
- лендінг для ПК;
- головна панель;
- особисті, спільні, парні й лімітовані квести;
- створення власних квестів;
- спільний магазин з обмеженим запасом;
- персональні й колективні покупки;
- колекція досягнень;
- профілі інших учасників зі статусом виконаного;
- сімейна стрічка активності;
- ручний калькулятор калорій без зовнішньої бази;
- PWA-режим.

Повна технічна схема: [ARCHITECTURE.md](ARCHITECTURE.md).


## GitHub і Cloudflare

- GitHub-репозиторій: `myHabbit`
- Cloudflare Worker: `myhabbit`
- GitHub Actions автоматично розгортає Worker із гілки `main`.

У **GitHub → Settings → Secrets and variables → Actions** додайте:

- `CLOUDFLARE_API_TOKEN`
- `CLOUDFLARE_ACCOUNT_ID`
- `TELEGRAM_BOT_TOKEN`

Telegram-токен не зберігається у файлах проєкту. Workflow передає його в Cloudflare як encrypted secret і автоматично налаштовує webhook, меню Mini App та команди бота.

## Telegram family session

У Telegram Mini App користувач бачить окрему форму з кодом сім'ї та PIN. Після серверної перевірки Telegram initData створюється або відновлюється сімейний профіль і локальна PWA-сесія.

## Local-first синхронізація v2.4

- Основний стан протягом дня зберігається локально на пристрої.
- Зміни формуються в один компактний денний пакет.
- Пакет відправляється при згортанні/закритті застосунку або при наступному запуску.
- Cloudflare Cron перевіряє сім'ї та після локальної 09:00 обробляє нові пакети один раз на день.
- В адмін-панелі є кнопка **«Оновити дані зараз»**. Вона спочатку надсилає локальні зміни адміністратора, а потім обробляє всі пакети, які вже надійшли від членів сім'ї.
- Ручне оновлення не бачить дані, які ще залишаються тільки на офлайн-пристрої іншого користувача.

Нові API:

- `POST /api/family/daily-submit`
- `GET /api/admin/sync-status`
- `POST /api/admin/process-now`


## Скидання профілю та Telegram-сесії (v2.5)

- У профілі є кнопка **«Скинути сесію»**. Вона вимагає сімейний PIN, від’єднує Telegram, анулює серверний токен і очищує localStorage, sessionStorage, IndexedDB, Cache Storage та Service Worker на пристрої.
- В адмін-панелі є блок **«Скидання користувача»**. Адміністратор обирає учасника, вводить сімейний PIN і додатково пише `СКИНУТИ`.
- Сервер обнуляє XP, монети, серію, навички, досягнення і статистику, видаляє очікуючі денні пакети, від’єднує Telegram та робить усі попередні сесії користувача недійсними.
- PWA-кеш переведено на нову версію, щоб Telegram Mini App отримував актуальні `app.js` і `styles.css`.


## JSON-бібліотека контенту (v2.6)

У `public/content/` вмонтовано локальну бібліотеку з 8 500 елементів: 5 500 унікальних квестів, 700 щоденних справ, 350 щотижневих справ, 450 челенджів, 1 200 досягнень і 300 предметів магазину. Каталог роздається як статичні файли Cloudflare Assets, кешується браузером і не витрачає API-запити Worker. Інтерфейс завантажує компактну активну вибірку, а повні JSON-файли залишаються доступними для подальшої ротації та серверної перевірки нагород.
